UISplitViewController教程：入门

了解如何将iOS应用程序拆分为两个部分，并在此UISplitViewController教程的每一侧显示视图控制器。

更新说明: Adam Rush为Xcode 11，iOS 13和Swift 5更新了本教程.Brad Johnson写了原文。

应用程序通常需要提供拆分视图以提供整洁的导航模型。 这方面的一个例子是Mail.app，它在iPad上使用左侧有文件夹列表的分割视图，然后是右侧选定的邮件项目。 Apple为我们构建了一个非常方便的视图控制器，称为UISplitViewController，它可以直接回到iPad的低端。 在这个UISplitViewController教程中，您将学习如何驯服它！ 此外，自iOS 8起，分割视图控制器可在iPad和iPhone上运行。

在本教程中，您将从头开始创建一个通用应用程序，它使用Split View Controller来显示Math Ninja中的怪物列表。 数学忍者是由Razeware团队自己开发的游戏！: ]

您将使用Split View Controller来处理导航和显示。 它适用于iPhone和iPad。

注意: 本教程重点介绍Split View Controller。 您应该已经熟悉了首先创建iOS应用程序的基础知识，例如AutoLayout和Storyboards。

入门

单击File ▸ New ▸ Project….在Xcode中创建一个新项目。 选择iOS ▸ Application ▸ Single View App模板。

将项目命名为MathMonsters。 将语言保持为迅捷。 将用户界面设置为Storyboard。 取消选中所有复选框。 然后单击“下一步”完成项目的创建。

虽然您可以使用Master-Detail App模板作为起点，但您将从头开始使用Single View App模板。 这将使您更好地了解UISplitViewController的工作原理。 在将来的项目中使用UISplitViewController时，这些知识将非常有用。

删除Storyboard中的默认初始视图控制器场景。 同时从项目导航器中删除ViewController.swift，确保在询问时选择Move to Trash。

这将为您的Storyboard添加几个元素:

  • Split View Controller: 此拆分视图将包含应用程序的其余部分，并且是应用程序的根目录。

  • Navigation Controller: 此UINavigationController将是主视图控制器的根视图。 这是拆分视图的左侧窗格，当在iPad上或在较大的iPhone（如iPhone 8 Plus）上横向显示时。

    在Split View Controller中，您将看到导航控制器具有称为主视图控制器的关系segue。 这允许您在主视图控制器中创建整个导航层次结构，而无需影响Detail View Controller。

  • View Controller: 这将最终显示所有怪物的详细信息。 如果查看Split View Controller，您将看到视图控制器具有称为Detail View Controller的关系segue:

  • Table View Controller: 这是主UINavigationController的根视图控制器。 它最终将显示怪物列表。

注意: Xcode会警告您表视图的原型单元缺少重用标识符。 暂时不要担心。 你很快就会解决它。

您将在分割视图控制器的左侧看到一个箭头。 这告诉你它是这个Storyboard的初始视图控制器。
