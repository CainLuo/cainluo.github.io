写完这个代码后, 细心的朋友可能会问到”为什么我们需要一个只发出一个.next事件的Observable对象?”

现在我们花几分钟回想一下我们之前学到的知识, 比如我们可以创建一个Observable对象使用以下的操作符:

  • Observable.never(): 创建一个Observable序列, 该序列永远不发出任何元素.
  • Observable.just(_:): 发出一个元素和一个.completed事件
  • Observable.empty(): 不发出元素, 但会发出一个.completed事件.
  • Observable.error(_): 不发出元素, 只发出一个.error事件.

如上面所说的, Observable可以生成0个或者是多个的.next事件的组合, 可能以.completed或.error结束.

在PhotoWriter的情况下, 我们只对一个事件感兴趣, 因为这种保存操作只会执行一次, 成功则是.next + .completed, 失败就发出.error.

这个时候我们就会想到, 是不是有更好的选择呢? 比如Single.

在实践中使用RxSwift的特性

在前面的第二章里, 我们了解了RxSwift的特性, 也就是Observable的特殊实现的方法, 在某些特殊的情况下使用是非常方便的.

在本章的工程里, 我们需要使用到这些特性, 现在就让我们来回顾一下, 先从Single开始:

Single

正如我们在第二章所了解到的, Single是一种特殊的Observable, 它表示一个序列, 并且该序列只能发出一次.success(Value)或.error事件, 我们如果查看了源码, 就会发现.success其实只是.next + .completed的组合.

这种特殊的序列在保存文件, 下载文件, 从磁盘加在数据或者是在异步操作中产生的数据都非常的有用, 我们可以以下面两个不同的Single示例进行分类:

  1. 封装一个在成功时只发出一个元素的操作, 就像本章的工程里的PhotoWriter.save(_). 我们可以直接创建Single, 而不是使用一个Observable对象, 在后面的挑战中, 我们会改造一下该方法.
  2. 为了更好的表达序列中使用单个元素的意图, 并且如果序列中发出多个元素, 那么订阅者则会收到错误事件.

如果我们要实现这个功能, 可以在任意订阅的Observable对象中使用.asSingle(), 将该Observable转换为Single对象, 等下你可以试试.

Maybe

Maybe与Single非常的相似, 唯一的区别就是在成功完成的时候, Observable对象不一定会发出值.

如果我们继续以工程为例, 可以试想一下使用Maybe这个操作符, 应用程序将照片保存到相册中, 我们将图片的ID保存在UserDefault中, 每次使用该ID来”打开”相册, 并插入一张照片, 我们可以设一个open(albumId:) -> Maybe<String>的方法处理以下的情况:

  • 如果该ID存在于相册, 只需要发出.completed事件.
  • 如果用户在删除相册的同事又创建了一个新相册, 并使用新ID发出.next事件, 呐我们就可以将该ID保存在UserDefault中.
  • 如果出现了错误, 我们根本访问不到相册, 那么就发出.error事件.

就和其他特性一样, 我们也可以使用常规的Observable对象来实现相同的功能, 但我们在编写代码时可能就会需要写更多的代码了.

与Single一样, 我们可以直接使用Maybe.create({ … })来创建一个Maybe对象, 也可以通过.asMaybe()来转换任意的Observable对象.

Completable

最后要介绍的一个特性就是Completable, 该Observable对象只允许在处理订阅之前发出一个.completed或.error事件.

我们可以使用ignoreElements()操作符将一个Observable序列转换成Completable, 这种情况就会忽略所有的事件, 只发出一个.completed或.error事件, 这也是Completable所需要的.

我们还可以使用Completable.create { … }创建一个完整的序列, 这里的代码和创建其他的Observable或者是特性非常的相似.

打开PhotoWriter.swift并将save(_)的返回类型更改为Single, 然后用Single.create替换掉常规Observable的创建.

这样子我们就可以替换掉一大部分的Xcode错误提示, 最后一件事要注意的是: Observable.create接收一个Observable作为参数, 这样我们就可以发出多个值和/或终止时间. Single.create接收一个闭包作为参数, 我们只能使用该闭包发出一个.success(T)或.error(E).

接下来就看你了, 记住参数是一个闭包, 不是一个Observable对象, 所以我们需要这样子调用它: single(.success(id)).

挑战2: 给自顶一个Observable对象添加一个警告弹框

打开MainViewController.swift, 并滑动到文件的地步, 找到showMessage(_: description:)方法.

该方法是在屏幕上显示一个UIAlertViewController, 当用户点击关闭按钮时将会运行一个回调, 然后UIAlertViewController就会消失. 这和我们之前写的PHPhotoLibrary.performChanges(_)所要做的工作不是很像吗?

我们现在要重写这个部分的逻辑, 下面是需求:

  • 向UIViewController添加一个Extension方法, 在屏幕上显示一个带有标题和消息的UIAlertViewController, 并且返回一个Completable.
  • 添加一个关闭按钮, 允许用户关闭该UIAlertViewController.
  • 在取消订阅时UIAlertViewController也将关闭, 这样子就不会有多余的弹框了.

最后, 使用新的completable从showMessage(_:description:)中显示弹框.
